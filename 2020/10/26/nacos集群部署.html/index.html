  • 服务发现和服务运行状况检查

    Nacos支持基于DNS和基于RPC（Dubbo / gRPC）的服务发现。服务提供商向本机 ，OpenAPI 或专用代理 注册服务后，使用者可以使用DNS 或HTTP 查找服务。

    Nacos提供实时运行状况检查，以防止服务将请求发送到不正常的主机或服务实例。Nacos支持传输层（PING或TCP）运行状况检查和应用程序层（例如HTTP，Redis，MySQL和用户定义的协议）运行状况检查。对于复杂云和网络拓扑（例如VPC，边缘服务等）的运行状况检查，Nacos提供代理模式和服务器模式运行状况检查。Nacos还提供统一的服务运行状况仪表板，以帮助您管理服务的可用性和流量。

  • 动态配置管理

    动态配置服务使您可以在所有环境中以集中，外部化和动态的方式管理所有应用程序和服务的配置。

    动态配置消除了在更新配置时重新部署应用程序和服务的需要。

    配置的集中管理使您更方便地实现无状态服务和按需弹性扩展服务实例。

    Nacos提供了易于使用的UI TODO ，可帮助您管理所有应用程序或服务的配置。它提供了一些现成的功能，包括配置版本跟踪，canary / beta版本，配置回滚和客户端配置更新状态跟踪，以确保安全并控制配置更改的风险。

  • 动态DNS服务

    支持加权路由的动态DNS服务使您可以更轻松地在数据中心内的生产环境中实施中间层负载平衡，灵活的路由策略，流量控制和简单的DNS解析服务。动态DNS服务使您更容易实现基于DNS的服务发现。

    Nacos提供了一些简单的DNS API TODO， 供您管理DNS域名和IP。

  • 服务治理和元数据管理

    Nacos允许您从微服务平台构建器的角度管理所有服务和元数据。这包括管理服务描述，生命周期，服务静态依赖关系分析，服务运行状况，服务流量管理，路由和安全规则，服务SLA和一线指标。

1.0 三台数据库配置

1[root@localhost ~]# vim /etc/my.cnf
2[mysqld]
3log-bin=mysql-bin # 非必需
4server-id=1　　　　# 必需
5master的server-id为1 那么slave1的server-id就是2 然后以此类推
6[root@localhost ~]# systemctl restart mysqld

2.0 单实例版nacos集群

  • 下载地址

  • 这里使用源代码方式部署

  • 集群最少3节点以上(采用端口号方式)

  • nacos1.0.0没有集群功能

  • JDK8+

  • maven3.2+

2.1 安装nacos

1[root@localhost ~]# tar -zxf nacos-server-1.3.2.tar.gz
2[root@localhost ~]# mv nacos /usr/local/nacos
3[root@localhost ~]# mv nacos /usr/local/nacos1
4[root@localhost ~]# mv nacos /usr/local/nacos2

2.2 修改nacos端口号

1# 默认不用修改nacos 直接修改nacos1和nacos2
2[root@localhost bin]# vim /usr/local/nacos1/conf/application.properties
3server.port=8849 # 更改为8849
4[root@localhost bin]# vim /usr/local/nacos2/conf/application.properties
5server.port=8850 # 更改为8850

2.3 配置集群文件

1[root@localhost nacos]# cd /usr/local/nacos/conf/
2[root@localhost conf]# cp cluster.conf.example cluster.conf // 三台机器都这么执行
3[root@localhost conf]# vim cluster.conf
4# 采用这种不同端口号来充当nacos实例
5192.168.20.10:8848
6192.168.20.10:8849
7192.168.20.10:8850

2.4 修改启动脚本

1[root@bogon bin]# vim /usr/local/nacos/bin/startup.sh
2JAVA_OPT="${JAVA_OPT} -server -Xms1g -Xmx1g # 修改启动内存为

2.5 启动nacos集群

1[root@localhost conf]# cd /usr/local/nacos/bin/
2sh startup.sh -p embedded  //三个配置全部启动
3# -p embedded # 表示采用内置数据源
4[root@localhost bin]# tail -f ../logs/nacos.log 

1# 启动成功提示
22020-10-18 22:13:26,644 INFO Nacos started successfully in cluster mode. use embedded storage

1访问：http://192.168.20.10:8848/nacos/index.html

3.0 多实例版nacos集群

1# 三台全部修改cluster.conf配置文件
2192.168.20.10:8848
3192.168.20.21:8848
4192.168.20.22:8848

1sh startup.sh -p embedded # 启动

1http://192.168.20.21:8848/nacos/index.html

3.1 nacos连接mysql存储源

1# 三台配置文件都修改
2[root@bogon nacos]# vim conf/application.properties
3spring.datasource.platform=mysql
4db.num=1  # 数据库个数
5db.url.0=jdbc:mysql://192.168.20.22:3306/nacos_config?characterEncoding=utf8&connectTimeout=1000&socketTimeout=3000&autoReconnect=true&useUnicode=true&useSSL=false&serverTimezone=UTC # 此处填写mysql的ip
6db.user=root  # 用户
7db.password=xxx  # 此为mysql的密码

3.2 导入mysql数据

1mysql> CREATE DATABASE nacos_config; # 先进入mysql创建一个库
2[root@bogon conf]# mysql -u root -p nacos_config < nacos-mysql.sql  //把nacos的表结构导入进去
3[root@bogon bin]# ./startup.sh 启动nacos
